Internal audit for Socialism Destroyer (Liberty Argument Engine). Primary purpose constraint: steelman → data-backed dismantle; free; no-account; no ad hominem.

## 1. Tech stack & architecture

| Layer | Finding |
|-------|---------|
| Framework | Flutter 3.x single codebase → web, iOS, Android, desktop |
| State | Riverpod |
| Routing | go_router (hash-style web routes `#/…`) |
| Offline | Hive (progress, notes, debate history); bundled knowledge assets |
| Hosting | Cloudflare Pages project `socialism-destroyer` |
| Canonical URL | https://destroyer.jonbailey.xyz |
| Deploy | `tools/publish-web.ps1` → `flutter build web` + `wrangler pages deploy` |
| Rendering | Client-side SPA (Flutter CanvasKit); shell HTML + `main.dart.js` |
| KB | Versioned JSON seeds + books under `assets/data/v2/` |

### Performance profile (pre-change)

- `build/web/main.dart.js` ≈ **4.3 MB** uncompressed (typical Flutter web cost).
- Books lazy-loaded from assets / knowledge CDN paths.
- Service worker present (`flutter_service_worker.js`) for caching.
- SEO shell is static; claim/book content is primarily in-app (SPA limitation for pure HTML crawlers — mitigated by sitemap + `llms.txt` + deep links).

**Targets:** Keep first paint shell navy/gold loader; avoid shipping API secrets in web build (`.env.web.publish`).
